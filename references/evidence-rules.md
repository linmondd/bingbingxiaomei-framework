# 证据与溯源规则

## A/B/C/D/U

- A：直接打开并核验作者一级来源全文或足够上下文，能够确认作者、标题、时间、正文语境与关键判断。
- B：一级来源 URL、状态 ID、标题、时间可由搜索索引与用户侧整理材料交叉验证，但页面无法稳定完整读取。搜索索引片段最高只能进入 B 级，不能升为 A。
- C：两个以上相互独立的二手来源一致复现；当前首版暂无 C 级。
- D：单一用户侧整理材料、播客转制或第三方知识库复现。
- U：没有可靠出处，或来源可能删除、锁定、仅在评论/图片/直播中出现。

证据等级描述的是“这句话能否归到作者及其原始语境”，不是观点是否正确。

## 来源状态

- `primary_verified`：一级来源已核验。
- `primary_indexed`：原帖可被索引和交叉确认。
- `secondary_reproduced`：由本地笔记或第三方材料复现。
- `unattributed_preserved`：概念被保留，但不能归为作者明确原话。
- `deleted_or_locked_possible`：存在删帖、锁帖或不可访问可能。
- `source_unavailable`：这是 Skill 自身方法约束，或来源当前不可得。

## 时间规则

- 优先记录原帖发布时间，精确到分钟时不得降为模糊日期。
- `first_seen` 是目前发现的最早时间，不等于理论首次形成时间。
- 历史回答必须区分“当时已知”“后来发展”和“今天回看”。
- 使用绝对日期，不以“今天、去年、最近”替代。

## 无来源保留

搜不到不等于不存在。对 U 级概念：

1. 保留概念名、候选含义、检索范围和最后检索日期。
2. 写入 `data/unresolved-sources.jsonl`。
3. 不构造原话、原帖时间或状态 ID。
4. 后续找到来源时升级，保留等级变化记录。

## 引用边界

- 只摘录足以识别观点的短句，避免复制整篇帖子。
- 第三方知识库只用于发现和交叉验证，不自动继承其所有归纳。
- 作者观点、整理者归纳、Agent 推断必须分别标识。
