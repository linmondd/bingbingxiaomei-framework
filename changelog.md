# Changelog

## v2.0.0 — 2026-07-04 · 全量升级

> 从 20 条手写 claims 到 10,764 条全量知识库的跨越。

### 架构升级

- **SKILL.md V2**：从扁平结构升级为五层 Surface 架构
  - Routing Surface — 加载条件与优先级
  - Contract Surface — 输入/输出契约与验收标准
  - Runtime Boundary — 按需加载知识文件与工具链
  - Safety Surface — 6 红线 + 4 黄线 + 必要声明
  - Expression DNA — 247 万字语料统计的风格指纹
- **回答工作流**：4-Step Agentic Protocol（分类→问诊→研究→回答）
- **Committee Mode**：投委会统一成员备忘录格式

### 全量数据采集

- API 全量采集 556 页，10,745 篇帖子（`bulk_ingest.py`）
- 10,745 条来源登记 + 10,764 条结构化观点卡
- S/A/B/C 四级质量分级，零丢弃，C 级保留为表达 DNA 素材

### 知识层

- **23 篇知识文件**：8 篇手写核心概念 + 13 篇 LLM 结构化初稿 + 2 篇框架文档
- 手写核心：三要素、亏钱效应、中央加杠杆、流动性三层、国运、金融三功能、产业映射、猪周期
- 所有 LLM 文件标注 AI 辅助状态
- `knowledge/INDEX.md` 与 `data/claims.jsonl` 双向关联

### 表达 DNA 分析

- `scripts/analyze_style.py`：四维指纹分析
  - 词汇：89,725 个独特词汇，Top 200 高频词统计
  - 句式：平均句长 41.1 字、疑问比例、转折词频率
  - 节奏：段落结构、编号列表率 23%
  - 视角：第一人称密度 17.5/万字、断言/模糊/数字密度

### 工程化

- **11 个管线脚本**：全量采集 → 质量分级 → DNA 分析 → LLM 提取 → 知识文件生成 → 校验
- `corpus/quality_check.py`：8 维度 SKILL.md 质量门（当前 97%）
- `query_evidence.py` 升级：`--fulltext` 全文检索原始帖子
- `references/research/01-distillation-log.md`：蒸馏过程完整记录
- 25 个单元测试，3,000 个非标准日期批量修复

### 安全修复

- 对抗性审查发现并修复 11 项风险（详见 `RELEASE_AUDIT.md`）
- 审计脚本中的敏感标记列表替换为通用模式
- 删库重建确保旧 commit 不可访问
- "窝"口癖替换为"冰冰小美"

### 表达 DNA 关键发现

| 维度 | 数据 | 含义 |
|------|------|------|
| 平均句长 | 41.1 字 | 中长句式，推理推进型 |
| 数字密度 | 204.9/万字 | 极高，数据驱动表达 |
| 断言词 | 2.5/万字 | 相对克制 |
